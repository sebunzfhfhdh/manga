Overview

The MangaWave API provides easy access to a wide range of manga data, including titles, chapters, covers, and more. Below are the available endpoints for interacting with the API.

API Endpoints

/manga

Get a list of manga based on specific query parameters. You can filter manga by various attributes such as title, type, year, and status.

Query Parameters

  • q (optional): The title of the manga you want to search for. Default is an empty string.
  • type (optional): The type of manga (e.g., 'Shonen', 'Seinen', etc.). Default is an empty string.
  • year (optional): The release year of the manga.
  • status (optional): The current status of the manga (e.g., 'Ongoing', 'Completed').
  • id (optional): The unique identifier of a specific manga.

Example Request

GET /manga?q=One%20Piece&type=Shonen&year=1999&status=Ongoing

Example Response

{
  "id": "2",
  "title": "One Piece",
  "type": "Manga",
  "year": "1999",
  "status": "Ongoing",
  "cover": "/images/https%3A%2F%2Fcdn.readdetectiveconan.com%2Ffile%2Fmangapill%2Fi%2F2.jpeg"
}

API Request Examples

Get Manga by Title

GET /manga?q=naruto

Searches for manga with the title "Naruto". Example response:

{
  "id": "1",
  "title": "Naruto",
  "type": "Shonen",
  "year": "1999",
  "status": "Completed",
  "cover": "/images/naruto_cover.jpg"
}

Error Handling

In case of an error, the API will return an error message along with the appropriate HTTP status code.

Common Errors

  • 400 Bad Request: Invalid or missing query parameters.
  • 404 Not Found: No manga found matching the given parameters.
  • 500 Internal Server Error: An unexpected error occurred on the server.

Example Error Response

{
  "error": "Invalid parameter",
  "message": "The 'type' parameter is invalid."
}
